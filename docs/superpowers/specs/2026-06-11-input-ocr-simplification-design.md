# 入力画面シンプル化と経費OCR 設計

## 目的

入力画面から稼働日下のプリセットボタンを外し、収入入力はUber売上の手入力に集中させる。経費入力は手動入力とレシートOCR入力の2通りを用意し、OCR結果から金額・カテゴリ・メモをフォームへ反映する。

## 方針

- 収入モードではレシート画像入力を表示しない。
- 経費モードではカテゴリ選択、金額入力、メモ入力を残す。
- 経費モードに「レシートを読み取る」ファイル入力を追加し、画像選択後に同期OCRを実行する。
- OCRはCloudflare AI Gatewayを使わず、Gemini Developer APIを`GEMINI_API_KEY`で直接呼ぶ。
- Google Searchなど追加課金につながるtoolは使わない。
- OCRが失敗しても手動入力は続けられる。

## OCRレスポンス

APIは`POST /api/receipts/analyze`を追加する。`multipart/form-data`の`file`を受け取り、以下を返す。

```json
{
  "amount": 1234,
  "date": "2026-06-11",
  "storeName": "ENEOS",
  "categoryCode": "601",
  "memo": "ENEOS レギュラーガソリン",
  "confidence": 0.86
}
```

## カテゴリ分類

確定申告で扱いやすい勘定科目へ集約する。

- ガソリン、軽油、充電、駐車場、洗車、高速道路: `601` 車両費
- スマホ、携帯、通信、SIM: `603` 通信費
- 配達バッグ、文具、備品、消耗品: `604` 消耗品費
- 保険: `605` 損害保険料
- 修理、メンテナンス、タイヤ: `606` 修繕費
- 判定不能: `699` 雑費

## 検証

- UIテストでプリセット非表示、収入レシート非表示、経費手動保存、OCR反映を確認する。
- サーバーテストでOCRテキストからカテゴリ分類ができることを確認する。
- lint、typecheck、unit test、build、ブラウザ確認を実行する。
